# Error Handling

Rust tidak memiliki exceptions. Sebagai gantinya, Rust menggunakan tipe `Result` dan `Option` untuk menangani error secara eksplisit dan aman.

## panic! vs Recoverable Errors

Ada dua kategori error di Rust:

- **Unrecoverable**: program berhenti total — gunakan `panic!`
- **Recoverable**: program bisa menangani — gunakan `Result` atau `Option`

```rust
fn main() {
    // panic! — program langsung berhenti
    // panic!("Something went terribly wrong!");

    // Akses index out of bounds juga panic
    let v = vec![1, 2, 3];
    // let x = v[10]; // panic: index out of bounds
    println!("{:?}", v);
}
```

## Result Enum

`Result` merepresentasikan operasi yang bisa berhasil atau gagal:

```rust
use std::fs::File;
use std::io::Read;

fn main() {
    // Result<T, E> = Ok(T) | Err(E)
    let file_result = File::open("hello.txt");

    match file_result {
        Ok(file) => println!("File opened: {:?}", file),
        Err(error) => println!("Error: {}", error),
    }
}
```

### unwrap dan expect

Shortcut yang panic jika hasilnya Err:

```rust
use std::fs::File;

fn main() {
    // unwrap — panic dengan pesan default
    // let f = File::open("hello.txt").unwrap();

    // expect — panic dengan pesan custom
    // let f = File::open("hello.txt").expect("Gagal membuka file");

    // Lebih aman: gunakan match atau ?
    let result = "42".parse::<i32>();
    let number = result.unwrap_or(0); // default value jika Err
    println!("Number: {}", number);

    let number2 = "abc".parse::<i32>().unwrap_or_default(); // 0 untuk i32
    println!("Number2: {}", number2);
}
```

## Option Enum

`Option` merepresentasikan nilai yang mungkin ada atau tidak:

```rust
fn find_first_even(numbers: &[i32]) -> Option<i32> {
    for &n in numbers {
        if n % 2 == 0 {
            return Some(n);
        }
    }
    None
}

fn main() {
    let nums = vec![1, 3, 5, 8, 9];

    match find_first_even(&nums) {
        Some(n) => println!("First even: {}", n),
        None => println!("No even number found"),
    }

    // Method-method berguna pada Option
    let x: Option<i32> = Some(42);
    let y: Option<i32> = None;

    println!("map: {:?}", x.map(|v| v * 2));       // Some(84)
    println!("unwrap_or: {}", y.unwrap_or(0));       // 0
    println!("is_some: {}", x.is_some());            // true
    println!("is_none: {}", y.is_none());            // true

    // and_then (flatmap)
    let result = x.and_then(|v| {
        if v > 10 { Some(v) } else { None }
    });
    println!("and_then: {:?}", result); // Some(42)
}
```

## Operator ? — Propagasi Error

Operator `?` secara otomatis mengembalikan error ke caller:

```rust
use std::fs::File;
use std::io::{self, Read};

fn read_file_content(path: &str) -> Result<String, io::Error> {
    let mut file = File::open(path)?;  // Jika Err, langsung return Err
    let mut content = String::new();
    file.read_to_string(&mut content)?; // Sama
    Ok(content)
}

// Bisa disingkat lagi:
fn read_file_short(path: &str) -> Result<String, io::Error> {
    std::fs::read_to_string(path)
}

fn main() {
    match read_file_content("test.txt") {
        Ok(content) => println!("Content: {}", content),
        Err(e) => println!("Error: {}", e),
    }
}
```

### Chaining dengan ?

```rust
use std::num::ParseIntError;

fn parse_and_double(s: &str) -> Result<i32, ParseIntError> {
    let n = s.trim().parse::<i32>()?;
    Ok(n * 2)
}

fn main() {
    println!("{:?}", parse_and_double("21"));   // Ok(42)
    println!("{:?}", parse_and_double("abc"));  // Err(...)
}
```

## Custom Error Types

### Dengan Display dan Error Traits

```rust
use std::fmt;
use std::io;
use std::num::ParseIntError;

#[derive(Debug)]
enum AppError {
    IoError(io::Error),
    ParseError(ParseIntError),
    ValidationError(String),
}

impl fmt::Display for AppError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            AppError::IoError(e) => write!(f, "IO error: {}", e),
            AppError::ParseError(e) => write!(f, "Parse error: {}", e),
            AppError::ValidationError(msg) => write!(f, "Validation error: {}", msg),
        }
    }
}

impl std::error::Error for AppError {}

// From trait — memungkinkan konversi otomatis dengan ?
impl From<io::Error> for AppError {
    fn from(e: io::Error) -> Self {
        AppError::IoError(e)
    }
}

impl From<ParseIntError> for AppError {
    fn from(e: ParseIntError) -> Self {
        AppError::ParseError(e)
    }
}

fn process_file(path: &str) -> Result<i32, AppError> {
    let content = std::fs::read_to_string(path)?; // io::Error -> AppError
    let number: i32 = content.trim().parse()?;      // ParseIntError -> AppError

    if number < 0 {
        return Err(AppError::ValidationError(
            "Number must be positive".to_string(),
        ));
    }

    Ok(number * 2)
}

fn main() {
    match process_file("number.txt") {
        Ok(result) => println!("Result: {}", result),
        Err(e) => println!("Error: {}", e),
    }
}
```

### thiserror Crate — Derive Macro

`thiserror` mengurangi boilerplate untuk custom errors:

```rust
// Cargo.toml: thiserror = "1"

use thiserror::Error;

#[derive(Error, Debug)]
enum DataError {
    #[error("File not found: {path}")]
    NotFound { path: String },

    #[error("Invalid format: {0}")]
    InvalidFormat(String),

    #[error("IO error")]
    Io(#[from] std::io::Error),

    #[error("Parse error")]
    Parse(#[from] std::num::ParseIntError),
}

fn load_data(path: &str) -> Result<Vec<i32>, DataError> {
    if !std::path::Path::new(path).exists() {
        return Err(DataError::NotFound {
            path: path.to_string(),
        });
    }

    let content = std::fs::read_to_string(path)?; // auto-convert via #[from]

    let numbers: Result<Vec<i32>, _> = content
        .lines()
        .map(|line| line.trim().parse::<i32>())
        .collect();

    numbers.map_err(|e| DataError::Parse(e))
}
```

### anyhow Crate — Untuk Aplikasi

`anyhow` cocok untuk aplikasi (bukan library) — menyederhanakan error handling:

```rust
// Cargo.toml: anyhow = "1"

use anyhow::{Context, Result, bail};

fn read_config(path: &str) -> Result<String> {
    let content = std::fs::read_to_string(path)
        .context(format!("Failed to read config from {}", path))?;
    Ok(content)
}

fn parse_port(config: &str) -> Result<u16> {
    let port: u16 = config
        .trim()
        .parse()
        .context("Failed to parse port number")?;

    if port < 1024 {
        bail!("Port must be >= 1024, got {}", port);
    }

    Ok(port)
}

fn main() -> Result<()> {
    let config = read_config("config.txt")?;
    let port = parse_port(&config)?;
    println!("Server running on port {}", port);
    Ok(())
}
```

## Kapan Pakai Mana?

| Situasi | Gunakan |
|---------|---------|
| Nilai yang mungkin kosong | `Option` |
| Operasi yang bisa gagal | `Result` |
| Bug yang tidak seharusnya terjadi | `panic!` / `unwrap` |
| Library — error types spesifik | `thiserror` |
| Aplikasi — simple error handling | `anyhow` |
| Prototype / quick script | `unwrap()` / `expect()` |

## Studi Kasus: Config File Parser

```rust
use std::collections::HashMap;
use std::fmt;

#[derive(Debug)]
enum ConfigError {
    FileNotFound(String),
    IoError(std::io::Error),
    ParseError { line: usize, message: String },
    MissingKey(String),
}

impl fmt::Display for ConfigError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            ConfigError::FileNotFound(path) => write!(f, "Config file not found: {}", path),
            ConfigError::IoError(e) => write!(f, "IO error: {}", e),
            ConfigError::ParseError { line, message } => {
                write!(f, "Parse error at line {}: {}", line, message)
            }
            ConfigError::MissingKey(key) => write!(f, "Missing required key: {}", key),
        }
    }
}

impl std::error::Error for ConfigError {}

impl From<std::io::Error> for ConfigError {
    fn from(e: std::io::Error) -> Self {
        ConfigError::IoError(e)
    }
}

struct Config {
    values: HashMap<String, String>,
}

impl Config {
    fn from_str(content: &str) -> Result<Config, ConfigError> {
        let mut values = HashMap::new();

        for (i, line) in content.lines().enumerate() {
            let line = line.trim();

            // Skip empty lines and comments
            if line.is_empty() || line.starts_with('#') {
                continue;
            }

            let parts: Vec<&str> = line.splitn(2, '=').collect();
            if parts.len() != 2 {
                return Err(ConfigError::ParseError {
                    line: i + 1,
                    message: format!("Expected key=value format, got: {}", line),
                });
            }

            let key = parts[0].trim().to_string();
            let value = parts[1].trim().to_string();

            if key.is_empty() {
                return Err(ConfigError::ParseError {
                    line: i + 1,
                    message: "Empty key".to_string(),
                });
            }

            values.insert(key, value);
        }

        Ok(Config { values })
    }

    fn get(&self, key: &str) -> Result<&str, ConfigError> {
        self.values
            .get(key)
            .map(|s| s.as_str())
            .ok_or_else(|| ConfigError::MissingKey(key.to_string()))
    }

    fn get_or(&self, key: &str, default: &str) -> String {
        self.values
            .get(key)
            .cloned()
            .unwrap_or_else(|| default.to_string())
    }

    fn get_typed<T: std::str::FromStr>(&self, key: &str) -> Result<T, ConfigError> {
        let value = self.get(key)?;
        value.parse::<T>().map_err(|_| ConfigError::ParseError {
            line: 0,
            message: format!("Cannot parse '{}' for key '{}'", value, key),
        })
    }
}

fn main() {
    let config_text = "
# Database settings
db_host = localhost
db_port = 5432
db_name = myapp

# Server settings
server_port = 8080
debug = true
    ";

    match Config::from_str(config_text) {
        Ok(config) => {
            println!("DB Host: {}", config.get("db_host").unwrap());
            println!("DB Port: {}", config.get_typed::<u16>("db_port").unwrap());
            println!("Server Port: {}", config.get_typed::<u16>("server_port").unwrap());
            println!("Log Level: {}", config.get_or("log_level", "info"));

            // Error case
            match config.get("missing_key") {
                Ok(val) => println!("Value: {}", val),
                Err(e) => println!("Error: {}", e),
            }
        }
        Err(e) => println!("Failed to parse config: {}", e),
    }
}
```

### Soal 3 — Operator ? (Sedang)

Buat fungsi `parse_coords` yang menerima string "x,y" dan mengembalikan `Result` of tuple (f64, f64). Gunakan ? untuk propagasi error.

**Jawaban:**

```rust
use std::num::ParseFloatError;

#[derive(Debug)]
enum CoordError {
    InvalidFormat,
    ParseError(ParseFloatError),
}

impl From<ParseFloatError> for CoordError {
    fn from(e: ParseFloatError) -> Self {
        CoordError::ParseError(e)
    }
}

fn parse_coords(s: &str) -> Result<(f64, f64), CoordError> {
    let parts: Vec<&str> = s.split(',').collect();
    if parts.len() != 2 {
        return Err(CoordError::InvalidFormat);
    }

    let x: f64 = parts[0].trim().parse()?;
    let y: f64 = parts[1].trim().parse()?;

    Ok((x, y))
}

fn main() {
    println!("{:?}", parse_coords("3.14, 2.71")); // Ok((3.14, 2.71))
    println!("{:?}", parse_coords("abc, 2.0"));    // Err(ParseError(...))
    println!("{:?}", parse_coords("1.0"));          // Err(InvalidFormat)
}
```

### Soal 4 — Custom Error Type (Sedang)

Buat enum `UserError` dengan varian `InvalidName`, `InvalidAge(String)`, `InvalidEmail(String)`. Implementasikan Display. Buat fungsi `validate_user` yang memvalidasi name, age, email.

**Jawaban:**

```rust
use std::fmt;

#[derive(Debug)]
enum UserError {
    InvalidName,
    InvalidAge(String),
    InvalidEmail(String),
}

impl fmt::Display for UserError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            UserError::InvalidName => write!(f, "Name cannot be empty"),
            UserError::InvalidAge(msg) => write!(f, "Invalid age: {}", msg),
            UserError::InvalidEmail(msg) => write!(f, "Invalid email: {}", msg),
        }
    }
}

impl std::error::Error for UserError {}

fn validate_user(name: &str, age: i32, email: &str) -> Result<(), UserError> {
    if name.trim().is_empty() {
        return Err(UserError::InvalidName);
    }
    if age < 0 || age > 150 {
        return Err(UserError::InvalidAge(format!("{} is out of range", age)));
    }
    if !email.contains('@') {
        return Err(UserError::InvalidEmail(format!("{} missing @", email)));
    }
    Ok(())
}

fn main() {
    let cases = vec![
        ("Budi", 25, "budi@mail.com"),
        ("", 25, "budi@mail.com"),
        ("Andi", -5, "andi@mail.com"),
        ("Citra", 30, "citra.mail.com"),
    ];

    for (name, age, email) in cases {
        match validate_user(name, age, email) {
            Ok(()) => println!("Valid: {} ({}, {})", name, age, email),
            Err(e) => println!("Error: {}", e),
        }
    }
}
```

### Soal 5 — Error Conversion dengan From (Sedang)

Buat program yang membaca file, parse isinya sebagai JSON-like key=value, dan kembalikan HashMap. Handle kedua io::Error dan parse error.

**Jawaban:**

```rust
use std::collections::HashMap;
use std::fmt;

#[derive(Debug)]
enum KvError {
    Io(std::io::Error),
    Format(String),
}

impl fmt::Display for KvError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            KvError::Io(e) => write!(f, "IO: {}", e),
            KvError::Format(msg) => write!(f, "Format: {}", msg),
        }
    }
}

impl From<std::io::Error> for KvError {
    fn from(e: std::io::Error) -> Self {
        KvError::Io(e)
    }
}

fn parse_kv(content: &str) -> Result<HashMap<String, String>, KvError> {
    let mut map = HashMap::new();

    for line in content.lines() {
        let line = line.trim();
        if line.is_empty() {
            continue;
        }

        let parts: Vec<&str> = line.splitn(2, '=').collect();
        if parts.len() != 2 {
            return Err(KvError::Format(format!("Bad line: {}", line)));
        }

        map.insert(parts[0].trim().to_string(), parts[1].trim().to_string());
    }

    Ok(map)
}

fn main() {
    let data = "name = Budi\nage = 25\ncity = Jakarta";
    match parse_kv(data) {
        Ok(map) => {
            for (k, v) in &map {
                println!("{}: {}", k, v);
            }
        }
        Err(e) => println!("Error: {}", e),
    }
}
```

### Soal 6 — Retry Pattern (Sulit)

Buat fungsi `retry` yang menjalankan closure berulang kali (max N kali) sampai berhasil. Jika semua gagal, kembalikan error terakhir.

**Jawaban:**

```rust
fn retry<T, E, F>(max_attempts: u32, mut operation: F) -> Result<T, E>
where
    F: FnMut(u32) -> Result<T, E>,
{
    let mut last_err = None;

    for attempt in 1..=max_attempts {
        match operation(attempt) {
            Ok(value) => return Ok(value),
            Err(e) => last_err = Some(e),
        }
    }

    Err(last_err.unwrap())
}

fn main() {
    let result = retry(3, |attempt| {
        println!("Attempt {}", attempt);
        if attempt < 3 {
            Err(format!("Failed on attempt {}", attempt))
        } else {
            Ok("Success!")
        }
    });

    println!("Result: {:?}", result);
}
```

### Soal 7 — Multiple Error Types (Sulit)

Buat program yang membaca file berisi angka (satu per baris), parse semua angka, hitung rata-rata. Handle: file not found, parse errors, empty file.

**Jawaban:**

```rust
use std::fmt;

#[derive(Debug)]
enum CalcError {
    Io(std::io::Error),
    Parse { line: usize, value: String },
    EmptyData,
}

impl fmt::Display for CalcError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            CalcError::Io(e) => write!(f, "File error: {}", e),
            CalcError::Parse { line, value } => {
                write!(f, "Cannot parse '{}' at line {}", value, line)
            }
            CalcError::EmptyData => write!(f, "No data to process"),
        }
    }
}

impl From<std::io::Error> for CalcError {
    fn from(e: std::io::Error) -> Self {
        CalcError::Io(e)
    }
}

fn calculate_average(content: &str) -> Result<f64, CalcError> {
    let mut numbers = Vec::new();

    for (i, line) in content.lines().enumerate() {
        let line = line.trim();
        if line.is_empty() {
            continue;
        }

        let n: f64 = line.parse().map_err(|_| CalcError::Parse {
            line: i + 1,
            value: line.to_string(),
        })?;

        numbers.push(n);
    }

    if numbers.is_empty() {
        return Err(CalcError::EmptyData);
    }

    let sum: f64 = numbers.iter().sum();
    Ok(sum / numbers.len() as f64)
}

fn main() {
    let data = "10\n20\n30\n40\n50";

    match calculate_average(data) {
        Ok(avg) => println!("Average: {:.2}", avg),
        Err(e) => println!("Error: {}", e),
    }
}
```

### Soal 8 — Result Combinator Chain (Sedang)

Buat pipeline yang: parse string ke i32, validasi range 1-100, kalikan 2, konversi ke String. Semua dengan method chaining pada Result.

**Jawaban:**

```rust
fn process(input: &str) -> Result<String, String> {
    input
        .trim()
        .parse::<i32>()
        .map_err(|e| format!("Parse error: {}", e))
        .and_then(|n| {
            if (1..=100).contains(&n) {
                Ok(n)
            } else {
                Err(format!("{} is out of range 1-100", n))
            }
        })
        .map(|n| n * 2)
        .map(|n| format!("Result: {}", n))
}

fn main() {
    println!("{:?}", process("42"));   // Ok("Result: 84")
    println!("{:?}", process("150"));  // Err("150 is out of range 1-100")
    println!("{:?}", process("abc"));  // Err("Parse error: ...")
}
```
