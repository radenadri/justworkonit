# Smart Pointers

Smart pointers adalah struct yang berperilaku seperti pointer tapi memiliki metadata dan kemampuan tambahan. Di Rust, smart pointers mengimplementasikan trait `Deref` dan `Drop`.

## Box — Heap Allocation

`Box` menyimpan data di heap, berguna untuk recursive types dan large data:

```rust
fn main() {
    // Menyimpan di heap
    let b = Box::new(5);
    println!("b = {}", b);

    // Berguna untuk large data
    let large_array = Box::new([0u8; 1_000_000]);
    println!("Large array size on stack: {} bytes", std::mem::size_of_val(&large_array));
}
```

### Recursive Types dengan Box

```rust
#[derive(Debug)]
enum List {
    Cons(i32, Box<List>),
    Nil,
}

use List::{Cons, Nil};

fn main() {
    let list = Cons(1, Box::new(Cons(2, Box::new(Cons(3, Box::new(Nil))))));
    println!("{:?}", list);
}
```

## Deref Trait

`Deref` memungkinkan smart pointer berperilaku seperti reference biasa:

```rust
use std::ops::Deref;

struct MyBox<T>(T);

impl<T> MyBox<T> {
    fn new(x: T) -> MyBox<T> {
        MyBox(x)
    }
}

impl<T> Deref for MyBox<T> {
    type Target = T;

    fn deref(&self) -> &T {
        &self.0
    }
}

fn hello(name: &str) {
    println!("Hello, {}!", name);
}

fn main() {
    let x = 5;
    let y = MyBox::new(x);

    assert_eq!(5, *y); // Deref coercion: *y menjadi *(y.deref())

    // Deref coercion: MyBox<String> -> String -> &str
    let m = MyBox::new(String::from("Rust"));
    hello(&m);
}
```

## Drop Trait

`Drop` menentukan apa yang terjadi saat value keluar dari scope:

```rust
struct CustomSmartPointer {
    data: String,
}

impl Drop for CustomSmartPointer {
    fn drop(&mut self) {
        println!("Dropping: {}", self.data);
    }
}

fn main() {
    let a = CustomSmartPointer { data: String::from("first") };
    let b = CustomSmartPointer { data: String::from("second") };
    println!("Created smart pointers");

    // Manual drop (jarang dipakai)
    drop(a);
    println!("Dropped 'a' early");

    println!("End of main");
    // b di-drop otomatis di sini
}
```

## Rc — Reference Counting

`Rc` memungkinkan multiple ownership — data di-drop saat count mencapai 0:

```rust
use std::rc::Rc;

#[derive(Debug)]
struct Node {
    value: i32,
    children: Vec<Rc<Node>>,
}

fn main() {
    let shared_child = Rc::new(Node {
        value: 10,
        children: vec![],
    });

    println!("Reference count: {}", Rc::strong_count(&shared_child));

    let parent1 = Node {
        value: 1,
        children: vec![Rc::clone(&shared_child)],
    };
    println!("After parent1: {}", Rc::strong_count(&shared_child));

    let parent2 = Node {
        value: 2,
        children: vec![Rc::clone(&shared_child)],
    };
    println!("After parent2: {}", Rc::strong_count(&shared_child));

    println!("parent1: {:?}", parent1);
    println!("parent2: {:?}", parent2);
}
```

Catatan penting: `Rc::clone()` hanya menambah reference count — murah. Berbeda dengan `.clone()` yang deep copy.

## RefCell — Interior Mutability

`RefCell` memindahkan borrow checking dari compile time ke runtime:

```rust
use std::cell::RefCell;

fn main() {
    let data = RefCell::new(vec![1, 2, 3]);

    // Immutable borrow
    println!("Data: {:?}", data.borrow());

    // Mutable borrow
    data.borrow_mut().push(4);
    println!("After push: {:?}", data.borrow());

    // Runtime borrow checking
    {
        let _r1 = data.borrow();
        let _r2 = data.borrow(); // OK: multiple immutable borrows
        // let _r3 = data.borrow_mut(); // PANIC: can't borrow mut while immut exists
    }
}
```

## Rc + RefCell — Shared Mutable Data

Kombinasi paling umum untuk shared mutable state:

```rust
use std::cell::RefCell;
use std::rc::Rc;

#[derive(Debug)]
struct SharedList {
    items: Rc<RefCell<Vec<String>>>,
}

impl SharedList {
    fn new() -> Self {
        SharedList {
            items: Rc::new(RefCell::new(Vec::new())),
        }
    }

    fn add(&self, item: &str) {
        self.items.borrow_mut().push(item.to_string());
    }

    fn get_shared(&self) -> Rc<RefCell<Vec<String>>> {
        Rc::clone(&self.items)
    }

    fn print(&self) {
        println!("Items: {:?}", self.items.borrow());
    }
}

fn main() {
    let list = SharedList::new();
    list.add("item1");
    list.add("item2");

    // Share data
    let shared = list.get_shared();
    shared.borrow_mut().push("item3".to_string());

    list.print(); // ["item1", "item2", "item3"]
    println!("Ref count: {}", Rc::strong_count(&list.items));
}
```

## Weak References — Mencegah Reference Cycles

`Weak` reference tidak menambah strong count — mencegah memory leaks:

```rust
use std::cell::RefCell;
use std::rc::{Rc, Weak};

#[derive(Debug)]
struct TreeNode {
    value: i32,
    parent: RefCell<Weak<TreeNode>>,
    children: RefCell<Vec<Rc<TreeNode>>>,
}

fn main() {
    let leaf = Rc::new(TreeNode {
        value: 3,
        parent: RefCell::new(Weak::new()),
        children: RefCell::new(vec![]),
    });

    println!("Leaf parent: {:?}", leaf.parent.borrow().upgrade());

    let branch = Rc::new(TreeNode {
        value: 5,
        parent: RefCell::new(Weak::new()),
        children: RefCell::new(vec![Rc::clone(&leaf)]),
    });

    // Set parent (Weak reference — tidak menambah strong count)
    *leaf.parent.borrow_mut() = Rc::downgrade(&branch);

    println!("Leaf parent: {:?}", leaf.parent.borrow().upgrade().map(|p| p.value));
    println!("Branch strong: {}, weak: {}",
        Rc::strong_count(&branch),
        Rc::weak_count(&branch),
    );
}
```

## Kapan Pakai Mana?

| Smart Pointer | Kapan Dipakai |
|---------------|---------------|
| `Box` | Heap allocation, recursive types, trait objects |
| `Rc` | Multiple ownership (single-threaded) |
| `RefCell` | Interior mutability (runtime borrow check) |
| `Rc + RefCell` | Shared mutable data (single-threaded) |
| `Arc` | Multiple ownership (multi-threaded) — lihat chapter Concurrency |
| `Weak` | Mencegah reference cycles |

## Studi Kasus: Simple DOM Tree

```rust
use std::cell::RefCell;
use std::rc::{Rc, Weak};

type NodeRef = Rc<DomNode>;

#[derive(Debug)]
struct DomNode {
    tag: String,
    text: RefCell<Option<String>>,
    parent: RefCell<Weak<DomNode>>,
    children: RefCell<Vec<NodeRef>>,
}

impl DomNode {
    fn new(tag: &str) -> NodeRef {
        Rc::new(DomNode {
            tag: tag.to_string(),
            text: RefCell::new(None),
            parent: RefCell::new(Weak::new()),
            children: RefCell::new(Vec::new()),
        })
    }

    fn with_text(tag: &str, text: &str) -> NodeRef {
        Rc::new(DomNode {
            tag: tag.to_string(),
            text: RefCell::new(Some(text.to_string())),
            parent: RefCell::new(Weak::new()),
            children: RefCell::new(Vec::new()),
        })
    }

    fn append_child(parent: &NodeRef, child: &NodeRef) {
        *child.parent.borrow_mut() = Rc::downgrade(parent);
        parent.children.borrow_mut().push(Rc::clone(child));
    }

    fn set_text(&self, text: &str) {
        *self.text.borrow_mut() = Some(text.to_string());
    }

    fn render(&self, depth: usize) -> String {
        let indent = "  ".repeat(depth);
        let mut output = String::new();

        output.push_str(&format!("{}<{}>\n", indent, self.tag));

        if let Some(text) = &*self.text.borrow() {
            output.push_str(&format!("{}  {}\n", indent, text));
        }

        for child in self.children.borrow().iter() {
            output.push_str(&child.render(depth + 1));
        }

        output.push_str(&format!("{}</{}>\n", indent, self.tag));
        output
    }

    fn find_by_tag(&self, tag: &str) -> Vec<NodeRef> {
        let mut results = Vec::new();

        for child in self.children.borrow().iter() {
            if child.tag == tag {
                results.push(Rc::clone(child));
            }
            results.extend(child.find_by_tag(tag));
        }

        results
    }
}

fn main() {
    let html = DomNode::new("html");
    let body = DomNode::new("body");
    let h1 = DomNode::with_text("h1", "Hello DOM");
    let div = DomNode::new("div");
    let p1 = DomNode::with_text("p", "First paragraph");
    let p2 = DomNode::with_text("p", "Second paragraph");

    DomNode::append_child(&html, &body);
    DomNode::append_child(&body, &h1);
    DomNode::append_child(&body, &div);
    DomNode::append_child(&div, &p1);
    DomNode::append_child(&div, &p2);

    println!("{}", html.render(0));

    // Find all <p> tags
    let paragraphs = html.find_by_tag("p");
    println!("Found {} paragraphs", paragraphs.len());

    // Modify text
    p1.set_text("Updated first paragraph");
    println!("\nAfter update:");
    println!("{}", html.render(0));

    // Check parent
    if let Some(parent) = p1.parent.borrow().upgrade() {
        println!("p1's parent: <{}>", parent.tag);
    }
}
```

## Bank Soal

### Soal 1 — Box Basics (Mudah)

Buat recursive binary tree menggunakan Box. Implementasikan method `sum` yang menjumlahkan semua nilai.

**Jawaban:**

```rust
#[derive(Debug)]
enum Tree {
    Leaf(i32),
    Node(i32, Box<Tree>, Box<Tree>),
}

impl Tree {
    fn sum(&self) -> i32 {
        match self {
            Tree::Leaf(v) => *v,
            Tree::Node(v, left, right) => v + left.sum() + right.sum(),
        }
    }
}

fn main() {
    let tree = Tree::Node(
        1,
        Box::new(Tree::Node(
            2,
            Box::new(Tree::Leaf(4)),
            Box::new(Tree::Leaf(5)),
        )),
        Box::new(Tree::Leaf(3)),
    );

    println!("{:?}", tree);
    println!("Sum: {}", tree.sum()); // 1+2+4+5+3 = 15
}
```

### Soal 3 — RefCell Interior Mutability (Sedang)

Buat struct `Logger` yang bisa di-share (immutable) tapi tetap bisa menambah log entries.

**Jawaban:**

```rust
use std::cell::RefCell;

struct Logger {
    entries: RefCell<Vec<String>>,
}

impl Logger {
    fn new() -> Self {
        Logger { entries: RefCell::new(Vec::new()) }
    }

    fn log(&self, message: &str) {
        self.entries.borrow_mut().push(format!(
            "[{}] {}", self.entries.borrow().len() + 1, message
        ));
    }

    fn dump(&self) {
        for entry in self.entries.borrow().iter() {
            println!("{}", entry);
        }
    }

    fn count(&self) -> usize {
        self.entries.borrow().len()
    }
}

fn main() {
    let logger = Logger::new();
    logger.log("Application started");
    logger.log("Processing data");
    logger.log("Done");

    println!("Total entries: {}", logger.count());
    logger.dump();
}
```

### Soal 4 — Rc + RefCell (Sedang)

Buat shared counter yang bisa diincrement dari beberapa tempat.

**Jawaban:**

```rust
use std::cell::RefCell;
use std::rc::Rc;

type SharedCounter = Rc<RefCell<i32>>;

fn make_counter() -> SharedCounter {
    Rc::new(RefCell::new(0))
}

fn increment(counter: &SharedCounter) {
    *counter.borrow_mut() += 1;
}

fn get_value(counter: &SharedCounter) -> i32 {
    *counter.borrow()
}

fn main() {
    let counter = make_counter();

    let c1 = Rc::clone(&counter);
    let c2 = Rc::clone(&counter);

    increment(&c1);
    increment(&c2);
    increment(&counter);

    println!("Counter: {}", get_value(&counter)); // 3
    println!("Ref count: {}", Rc::strong_count(&counter)); // 3
}
```

### Soal 6 — Weak References (Sulit)

Buat graph node yang bisa memiliki parent (Weak) dan children (Rc). Demonstrasikan bahwa reference cycle tidak terjadi.

**Jawaban:**

```rust
use std::cell::RefCell;
use std::rc::{Rc, Weak};

struct GraphNode {
    id: u32,
    parent: RefCell<Weak<GraphNode>>,
    children: RefCell<Vec<Rc<GraphNode>>>,
}

impl GraphNode {
    fn new(id: u32) -> Rc<Self> {
        Rc::new(GraphNode {
            id,
            parent: RefCell::new(Weak::new()),
            children: RefCell::new(Vec::new()),
        })
    }

    fn add_child(parent: &Rc<GraphNode>, child: &Rc<GraphNode>) {
        *child.parent.borrow_mut() = Rc::downgrade(parent);
        parent.children.borrow_mut().push(Rc::clone(child));
    }
}

impl Drop for GraphNode {
    fn drop(&mut self) {
        println!("Dropping node {}", self.id);
    }
}

fn main() {
    let root = GraphNode::new(1);
    let child_a = GraphNode::new(2);
    let child_b = GraphNode::new(3);

    GraphNode::add_child(&root, &child_a);
    GraphNode::add_child(&root, &child_b);

    println!("Root strong: {}", Rc::strong_count(&root));
    println!("Child A strong: {}, root weak: {}",
        Rc::strong_count(&child_a),
        Rc::weak_count(&root),
    );

    // Parent access via Weak
    if let Some(parent) = child_a.parent.borrow().upgrade() {
        println!("Child A's parent: node {}", parent.id);
    }
} // All nodes dropped — no cycle!
```

### Soal 7 — Observer Pattern (Sulit)

Implementasikan observer pattern menggunakan Rc dan RefCell. EventEmitter yang bisa register/notify observers.

**Jawaban:**

```rust
use std::cell::RefCell;
use std::rc::Rc;

type Callback = Box<dyn Fn(&str)>;

struct EventEmitter {
    listeners: RefCell<Vec<Callback>>,
}

impl EventEmitter {
    fn new() -> Rc<Self> {
        Rc::new(EventEmitter {
            listeners: RefCell::new(Vec::new()),
        })
    }

    fn on(&self, callback: impl Fn(&str) + 'static) {
        self.listeners.borrow_mut().push(Box::new(callback));
    }

    fn emit(&self, event: &str) {
        for listener in self.listeners.borrow().iter() {
            listener(event);
        }
    }

    fn listener_count(&self) -> usize {
        self.listeners.borrow().len()
    }
}

fn main() {
    let emitter = EventEmitter::new();

    emitter.on(|event| println!("Logger: {}", event));
    emitter.on(|event| println!("Analytics: {}", event));
    emitter.on(|event| println!("Notification: {}", event));

    println!("Listeners: {}", emitter.listener_count());

    emitter.emit("user_login");
    println!();
    emitter.emit("purchase");
}
```
