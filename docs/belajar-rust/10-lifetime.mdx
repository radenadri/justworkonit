# Lifetime

Lifetime adalah mekanisme Rust untuk memastikan semua reference valid — tidak ada dangling reference. Compiler menggunakan lifetime annotations untuk memverifikasi bahwa data yang direferensikan masih hidup selama reference digunakan.

## Mengapa Lifetime Diperlukan?

```rust
// Ini TIDAK akan compile:
// fn longest(x: &str, y: &str) -> &str {
//     if x.len() > y.len() { x } else { y }
// }
// Error: missing lifetime specifier
// Compiler tidak tahu apakah return value mengacu ke x atau y

// Solusi: lifetime annotation
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}

fn main() {
    let s1 = String::from("long string");
    let result;
    {
        let s2 = String::from("xyz");
        result = longest(s1.as_str(), s2.as_str());
        println!("Longest: {}", result); // OK: s2 masih hidup
    }
    // println!("{}", result); // ERROR jika uncomment: s2 sudah drop
}
```

## Syntax Lifetime Annotation

Lifetime annotation ditulis dengan `'` diikuti nama (biasanya huruf kecil pendek):

```rust
// Reference dengan lifetime
// &'a i32        — reference dengan lifetime 'a
// &'a mut i32    — mutable reference dengan lifetime 'a

fn first_word<'a>(s: &'a str) -> &'a str {
    let bytes = s.as_bytes();
    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[..i];
        }
    }
    s
}

fn main() {
    let sentence = String::from("hello world");
    let word = first_word(&sentence);
    println!("First word: {}", word);
}
```

## Lifetime pada Struct

Struct yang menyimpan reference harus memiliki lifetime annotation:

```rust
#[derive(Debug)]
struct Excerpt<'a> {
    text: &'a str,
}

impl<'a> Excerpt<'a> {
    fn level(&self) -> i32 {
        3
    }

    // Return lifetime terikat ke &self (elision rule #3)
    fn announce(&self, prefix: &str) -> &str {
        println!("{}: {}", prefix, self.text);
        self.text
    }
}

fn main() {
    let novel = String::from("Call me Ishmael. Some years ago...");
    let first_sentence;

    {
        let i = novel.find('.').unwrap_or(novel.len());
        first_sentence = Excerpt { text: &novel[..i] };
    }

    println!("Excerpt: {:?}", first_sentence);
    println!("Level: {}", first_sentence.level());
}
```

## Lifetime Elision Rules

Rust memiliki 3 aturan elision sehingga kita tidak perlu menulis lifetime di setiap fungsi:

**Rule 1:** Setiap reference parameter mendapat lifetime sendiri.

**Rule 2:** Jika hanya ada satu input lifetime, itu diterapkan ke semua output references.

**Rule 3:** Jika ada `&self` atau `&mut self`, lifetime self diterapkan ke semua output references.

```rust
// Rule 1 & 2 — satu input reference
fn first_char(s: &str) -> &str {
    &s[..1]
}
// Sama dengan: fn first_char<'a>(s: &'a str) -> &'a str

// Rule 3 — method dengan &self
struct Config {
    name: String,
}

impl Config {
    fn get_name(&self) -> &str {
        &self.name
    }
    // Sama dengan: fn get_name<'a>(&'a self) -> &'a str
}

// Kapan HARUS tulis manual:
// - Lebih dari satu input reference dan return reference
// - Compiler tidak bisa infer mana lifetime yang tepat
fn longer<'a>(x: &'a str, y: &str) -> &'a str {
    // Return selalu dari x, jadi hanya x perlu lifetime 'a
    x
}

fn main() {
    let s = String::from("hello");
    println!("{}", first_char(&s));

    let cfg = Config { name: "app".to_string() };
    println!("{}", cfg.get_name());
}
```

## 'static Lifetime

`'static` berarti reference bisa hidup sepanjang program:

```rust
// String literal selalu 'static
let s: &'static str = "I live forever";

// Owned data yang di-leak juga 'static (jarang dipakai)
fn make_static() -> &'static str {
    let s = String::from("leaked");
    Box::leak(s.into_boxed_str())
}

fn main() {
    let s: &'static str = "hello";
    println!("{}", s);

    // Generic bound dengan 'static
    fn print_static<T: std::fmt::Display + 'static>(val: T) {
        println!("{}", val);
    }

    print_static(42);
    print_static("hello");
}
```

## Multiple Lifetimes

```rust
// Dua lifetime berbeda
fn first_or_second<'a, 'b>(first: &'a str, second: &'b str, use_first: bool) -> &'a str
where
    'b: 'a, // 'b outlives 'a
{
    if use_first { first } else { second }
}

// Struct dengan multiple lifetimes
#[derive(Debug)]
struct TwoRefs<'a, 'b> {
    x: &'a str,
    y: &'b str,
}

fn main() {
    let s1 = String::from("hello");
    let s2 = String::from("world");
    let refs = TwoRefs { x: &s1, y: &s2 };
    println!("{:?}", refs);
}
```

## Common Patterns dan Pitfalls

### Pattern: Return Owned Data Daripada Reference

```rust
// Kadang lebih mudah return owned data
fn format_name(first: &str, last: &str) -> String {
    // Return String (owned), bukan &str
    // Tidak perlu lifetime karena ownership berpindah
    format!("{} {}", first, last)
}

fn main() {
    let name = format_name("Budi", "Santoso");
    println!("{}", name);
}
```

### Pitfall: Dangling Reference

```rust
// TIDAK compile:
// fn dangle() -> &String {
//     let s = String::from("hello");
//     &s  // s di-drop di akhir fungsi, reference jadi dangling
// }

// Solusi: return owned value
fn no_dangle() -> String {
    let s = String::from("hello");
    s // ownership berpindah ke caller
}

fn main() {
    let s = no_dangle();
    println!("{}", s);
}
```

## Studi Kasus: Text Excerpt Finder

```rust
#[derive(Debug)]
struct Document<'a> {
    content: &'a str,
}

#[derive(Debug)]
struct SearchResult<'a> {
    line_number: usize,
    line: &'a str,
    start: usize,
    end: usize,
}

impl<'a> Document<'a> {
    fn new(content: &'a str) -> Self {
        Document { content }
    }

    fn search(&self, query: &str) -> Vec<SearchResult<'a>> {
        let mut results = Vec::new();

        for (line_num, line) in self.content.lines().enumerate() {
            if let Some(pos) = line.to_lowercase().find(&query.to_lowercase()) {
                results.push(SearchResult {
                    line_number: line_num + 1,
                    line: line.trim(),
                    start: pos,
                    end: pos + query.len(),
                });
            }
        }

        results
    }

    fn excerpt(&self, start: usize, length: usize) -> &'a str {
        let end = (start + length).min(self.content.len());
        &self.content[start..end]
    }

    fn word_count(&self) -> usize {
        self.content.split_whitespace().count()
    }

    fn lines(&self) -> Vec<&'a str> {
        self.content.lines().collect()
    }

    fn nth_line(&self, n: usize) -> Option<&'a str> {
        self.content.lines().nth(n)
    }
}

impl<'a> std::fmt::Display for SearchResult<'a> {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        write!(
            f,
            "Line {}: \"{}\" (position {}-{})",
            self.line_number, self.line, self.start, self.end
        )
    }
}

fn main() {
    let text = "\
Rust is a systems programming language.
It focuses on safety and performance.
The Rust compiler prevents many bugs at compile time.
Learning Rust can be challenging but rewarding.
Rust has a strong and welcoming community.";

    let doc = Document::new(text);

    println!("=== Document Stats ===");
    println!("Words: {}", doc.word_count());
    println!("Lines: {}", doc.lines().len());
    println!();

    println!("=== Search: 'rust' ===");
    let results = doc.search("rust");
    for result in &results {
        println!("{}", result);
    }
    println!();

    println!("=== Excerpt (0..50) ===");
    println!("{}", doc.excerpt(0, 50));
    println!();

    println!("=== Line 3 ===");
    if let Some(line) = doc.nth_line(2) {
        println!("{}", line);
    }
}
```

### Soal 2 — Struct dengan Lifetime (Mudah)

Buat struct `Highlight` yang menyimpan reference ke text dan posisi start/end. Implementasikan method `text_slice` yang mengembalikan potongan teks.

**Jawaban:**

```rust
#[derive(Debug)]
struct Highlight<'a> {
    source: &'a str,
    start: usize,
    end: usize,
}

impl<'a> Highlight<'a> {
    fn new(source: &'a str, start: usize, end: usize) -> Self {
        Highlight { source, start, end }
    }

    fn text_slice(&self) -> &'a str {
        &self.source[self.start..self.end]
    }
}

fn main() {
    let text = String::from("The quick brown fox");
    let highlight = Highlight::new(&text, 4, 9);
    println!("Highlighted: '{}'", highlight.text_slice()); // "quick"
}
```

### Soal 4 — Lifetime dengan Enum (Sedang)

Buat enum `Token` yang bisa menyimpan reference ke source string. Varian: `Word`, `Number`, `Symbol`.

**Jawaban:**

```rust
#[derive(Debug)]
enum Token<'a> {
    Word(&'a str),
    Number(&'a str),
    Symbol(&'a str),
}

fn tokenize<'a>(input: &'a str) -> Vec<Token<'a>> {
    let mut tokens = Vec::new();

    for word in input.split_whitespace() {
        if word.chars().all(|c| c.is_numeric()) {
            tokens.push(Token::Number(word));
        } else if word.chars().all(|c| c.is_alphabetic()) {
            tokens.push(Token::Word(word));
        } else {
            tokens.push(Token::Symbol(word));
        }
    }

    tokens
}

fn main() {
    let source = "hello 42 + world 99";
    let tokens = tokenize(source);

    for token in &tokens {
        match token {
            Token::Word(w) => println!("WORD: {}", w),
            Token::Number(n) => println!("NUM:  {}", n),
            Token::Symbol(s) => println!("SYM:  {}", s),
        }
    }
}
```

### Soal 5 — Lifetime Elision (Sedang)

Tentukan mana yang perlu lifetime annotation eksplisit dan mana yang tidak (compiler bisa infer). Tulis versi lengkap:

```rust
// a) fn get_first(s: &str) -> &str
// b) fn longest(a: &str, b: &str) -> &str
// c) fn push_str(s: &mut String, addition: &str)
```

**Jawaban:**

```rust
// a) TIDAK perlu — Rule 1 & 2 (satu input ref → output ref sama)
fn get_first(s: &str) -> &str {
    &s[..1]
}
// Lengkap: fn get_first<'a>(s: &'a str) -> &'a str

// b) PERLU — dua input refs, compiler tidak tahu mana yang dikembalikan
fn longest<'a>(a: &'a str, b: &'a str) -> &'a str {
    if a.len() >= b.len() { a } else { b }
}

// c) TIDAK perlu — tidak ada return reference
fn push_str(s: &mut String, addition: &str) {
    s.push_str(addition);
}

fn main() {
    println!("{}", get_first("hello"));
    println!("{}", longest("abc", "de"));

    let mut s = String::from("hello ");
    push_str(&mut s, "world");
    println!("{}", s);
}
```

### Soal 6 — Lifetime dan Trait (Sulit)

Buat trait `Summarizable` dengan method yang mengembalikan `&str`. Implementasikan untuk struct yang menyimpan reference.

**Jawaban:**

```rust
trait Summarizable<'a> {
    fn summary(&self) -> &'a str;
}

struct Article<'a> {
    headline: &'a str,
    body: &'a str,
}

impl<'a> Summarizable<'a> for Article<'a> {
    fn summary(&self) -> &'a str {
        self.headline
    }
}

struct Tweet<'a> {
    content: &'a str,
}

impl<'a> Summarizable<'a> for Tweet<'a> {
    fn summary(&self) -> &'a str {
        self.content
    }
}

fn main() {
    let headline = String::from("Breaking: Rust 2024!");
    let article = Article {
        headline: &headline,
        body: "Details here...",
    };

    let tweet_text = String::from("Loving Rust!");
    let tweet = Tweet { content: &tweet_text };

    println!("Article: {}", article.summary());
    println!("Tweet: {}", tweet.summary());
}
```

### Soal 7 — Iterator dengan Lifetime (Sulit)

Buat struct `SentenceIter` yang mengiterasi kalimat (dipisahkan titik) dari sebuah teks. Implementasikan Iterator trait.

**Jawaban:**

```rust
struct SentenceIter<'a> {
    remaining: &'a str,
}

impl<'a> SentenceIter<'a> {
    fn new(text: &'a str) -> Self {
        SentenceIter { remaining: text.trim() }
    }
}

impl<'a> Iterator for SentenceIter<'a> {
    type Item = &'a str;

    fn next(&mut self) -> Option<Self::Item> {
        if self.remaining.is_empty() {
            return None;
        }

        match self.remaining.find('.') {
            Some(pos) => {
                let sentence = self.remaining[..pos].trim();
                self.remaining = self.remaining[pos + 1..].trim();
                if sentence.is_empty() {
                    self.next()
                } else {
                    Some(sentence)
                }
            }
            None => {
                let sentence = self.remaining.trim();
                self.remaining = "";
                if sentence.is_empty() {
                    None
                } else {
                    Some(sentence)
                }
            }
        }
    }
}

fn main() {
    let text = "Hello world. Rust is great. Learning is fun.";

    for (i, sentence) in SentenceIter::new(text).enumerate() {
        println!("Sentence {}: {}", i + 1, sentence);
    }
}
```
