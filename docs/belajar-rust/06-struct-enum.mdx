# Struct, Enum & Pattern Matching

Struct dan enum adalah fondasi type system di Rust. Dengan struct, kita mendefinisikan data terstruktur. Dengan enum, kita merepresentasikan variasi. Dan dengan pattern matching, kita mengurai keduanya secara aman.

## Struct

Struct mengelompokkan data terkait dalam satu tipe.

### Struct Biasa

```rust
struct User {
    username: String,
    email: String,
    age: u32,
    active: bool,
}

fn main() {
    let user1 = User {
        username: String::from("budi"),
        email: String::from("budi@mail.com"),
        age: 25,
        active: true,
    };

    println!("User: {}, umur: {}", user1.username, user1.age);

    // Struct update syntax
    let user2 = User {
        username: String::from("andi"),
        ..user1
    };

    println!("User2 email: {}", user2.email);
}
```

### Tuple Struct

Tuple struct berguna ketika kita ingin memberi nama pada tipe tanpa menamai setiap field:

```rust
struct Color(u8, u8, u8);
struct Point(f64, f64, f64);

fn main() {
    let red = Color(255, 0, 0);
    let origin = Point(0.0, 0.0, 0.0);

    println!("Red: ({}, {}, {})", red.0, red.1, red.2);
    println!("Origin: ({}, {}, {})", origin.0, origin.1, origin.2);
}
```

### Unit Struct

Unit struct tidak memiliki field — berguna sebagai marker:

```rust
struct Marker;

fn main() {
    let _m = Marker;
}
```

## Impl Block — Method & Associated Function

Kita menambahkan behavior pada struct melalui `impl` block:

```rust
struct Rectangle {
    width: f64,
    height: f64,
}

impl Rectangle {
    // Associated function (constructor) — tanpa &self
    fn new(width: f64, height: f64) -> Rectangle {
        Rectangle { width, height }
    }

    fn square(size: f64) -> Rectangle {
        Rectangle { width: size, height: size }
    }

    // Method — menerima &self
    fn area(&self) -> f64 {
        self.width * self.height
    }

    fn perimeter(&self) -> f64 {
        2.0 * (self.width + self.height)
    }

    fn can_hold(&self, other: &Rectangle) -> bool {
        self.width > other.width && self.height > other.height
    }
}

fn main() {
    let rect = Rectangle::new(10.0, 5.0);
    let sq = Rectangle::square(4.0);

    println!("Area rect: {}", rect.area());
    println!("Perimeter rect: {}", rect.perimeter());
    println!("Rect holds square? {}", rect.can_hold(&sq));
}
```

Perbedaan method vs associated function: method menerima `&self` (atau `&mut self`, `self`), sedangkan associated function tidak — dipanggil dengan `Type::function()`.

## Enum

Enum merepresentasikan satu dari beberapa varian yang mungkin:

```rust
enum Direction {
    North,
    South,
    East,
    West,
}

fn describe(dir: Direction) {
    match dir {
        Direction::North => println!("Ke utara"),
        Direction::South => println!("Ke selatan"),
        Direction::East => println!("Ke timur"),
        Direction::West => println!("Ke barat"),
    }
}

fn main() {
    describe(Direction::North);
}
```

### Enum dengan Data

Setiap varian enum bisa menyimpan data berbeda:

```rust
enum Message {
    Quit,
    Echo(String),
    Move { x: i32, y: i32 },
    Color(u8, u8, u8),
}

impl Message {
    fn process(&self) {
        match self {
            Message::Quit => println!("Quit"),
            Message::Echo(text) => println!("Echo: {}", text),
            Message::Move { x, y } => println!("Move to ({}, {})", x, y),
            Message::Color(r, g, b) => println!("Color: ({}, {}, {})", r, g, b),
        }
    }
}

fn main() {
    let messages = vec![
        Message::Echo(String::from("hello")),
        Message::Move { x: 10, y: 20 },
        Message::Color(255, 128, 0),
        Message::Quit,
    ];

    for msg in &messages {
        msg.process();
    }
}
```

### Option Enum

`Option` adalah enum bawaan Rust untuk merepresentasikan nilai yang mungkin tidak ada — pengganti null:

```rust
fn find_item(items: &[&str], target: &str) -> Option<usize> {
    for (i, item) in items.iter().enumerate() {
        if *item == target {
            return Some(i);
        }
    }
    None
}

fn main() {
    let fruits = vec!["apel", "jeruk", "mangga"];

    match find_item(&fruits, "jeruk") {
        Some(index) => println!("Ditemukan di index {}", index),
        None => println!("Tidak ditemukan"),
    }
}
```

## Pattern Matching

### match Expression

`match` harus exhaustive — semua kemungkinan harus di-cover:

```rust
fn classify_number(n: i32) -> &'static str {
    match n {
        0 => "nol",
        1..=9 => "satu digit",
        10..=99 => "dua digit",
        100..=999 => "tiga digit",
        _ => "banyak digit",
    }
}

fn main() {
    println!("{}", classify_number(42));   // dua digit
    println!("{}", classify_number(7));    // satu digit
    println!("{}", classify_number(1000)); // banyak digit
}
```

### if let dan while let

Untuk kasus di mana kita hanya peduli satu pattern:

```rust
fn main() {
    let config_max: Option<u32> = Some(100);

    // if let — match satu pattern saja
    if let Some(max) = config_max {
        println!("Max: {}", max);
    }

    // while let — loop selama pattern cocok
    let mut stack = vec![1, 2, 3];

    while let Some(top) = stack.pop() {
        println!("Popped: {}", top);
    }
}
```

### Nested Pattern & Guard

```rust
enum Shape {
    Circle(f64),
    Rectangle(f64, f64),
}

fn describe_shape(shape: &Shape) {
    match shape {
        Shape::Circle(r) if *r > 10.0 => println!("Lingkaran besar, r={}", r),
        Shape::Circle(r) => println!("Lingkaran kecil, r={}", r),
        Shape::Rectangle(w, h) if (w - h).abs() < f64::EPSILON => {
            println!("Persegi, sisi={}", w)
        }
        Shape::Rectangle(w, h) => println!("Persegi panjang {}x{}", w, h),
    }
}

fn main() {
    describe_shape(&Shape::Circle(5.0));
    describe_shape(&Shape::Circle(15.0));
    describe_shape(&Shape::Rectangle(4.0, 4.0));
    describe_shape(&Shape::Rectangle(3.0, 7.0));
}
```

### Destructuring Struct dalam Pattern

```rust
struct Point {
    x: f64,
    y: f64,
}

fn quadrant(p: &Point) -> &'static str {
    match p {
        Point { x, y } if *x > 0.0 && *y > 0.0 => "Kuadran I",
        Point { x, y } if *x < 0.0 && *y > 0.0 => "Kuadran II",
        Point { x, y } if *x < 0.0 && *y < 0.0 => "Kuadran III",
        Point { x, y } if *x > 0.0 && *y < 0.0 => "Kuadran IV",
        _ => "Di sumbu",
    }
}

fn main() {
    let p = Point { x: 3.0, y: -2.0 };
    println!("{}", quadrant(&p));
}
```

## Studi Kasus: Sistem Inventory Sederhana

```rust
#[derive(Debug)]
enum Category {
    Electronics,
    Clothing,
    Food,
    Books,
}

#[derive(Debug)]
struct Product {
    name: String,
    category: Category,
    price: f64,
    stock: u32,
}

impl Product {
    fn new(name: &str, category: Category, price: f64, stock: u32) -> Product {
        Product {
            name: String::from(name),
            category,
            price,
            stock,
        }
    }

    fn is_available(&self) -> bool {
        self.stock > 0
    }

    fn apply_discount(&mut self, percent: f64) {
        self.price *= 1.0 - (percent / 100.0);
    }

    fn category_label(&self) -> &str {
        match &self.category {
            Category::Electronics => "Elektronik",
            Category::Clothing => "Pakaian",
            Category::Food => "Makanan",
            Category::Books => "Buku",
        }
    }
}

struct Inventory {
    products: Vec<Product>,
}

impl Inventory {
    fn new() -> Inventory {
        Inventory { products: Vec::new() }
    }

    fn add(&mut self, product: Product) {
        self.products.push(product);
    }

    fn find_by_name(&self, name: &str) -> Option<&Product> {
        self.products.iter().find(|p| p.name == name)
    }

    fn total_value(&self) -> f64 {
        self.products.iter()
            .map(|p| p.price * p.stock as f64)
            .sum()
    }

    fn by_category(&self, cat: &Category) -> Vec<&Product> {
        self.products.iter()
            .filter(|p| std::mem::discriminant(&p.category) == std::mem::discriminant(cat))
            .collect()
    }

    fn summary(&self) {
        println!("=== Inventory Summary ===");
        for product in &self.products {
            let status = if product.is_available() { "✓" } else { "✗" };
            println!(
                "[{}] {} - {} | Rp{:.0} | Stok: {}",
                status,
                product.name,
                product.category_label(),
                product.price,
                product.stock
            );
        }
        println!("Total value: Rp{:.0}", self.total_value());
    }
}

fn main() {
    let mut inv = Inventory::new();

    inv.add(Product::new("Laptop", Category::Electronics, 12_000_000.0, 5));
    inv.add(Product::new("Kaos", Category::Clothing, 150_000.0, 50));
    inv.add(Product::new("Nasi Goreng", Category::Food, 25_000.0, 0));
    inv.add(Product::new("Rust Book", Category::Books, 500_000.0, 10));

    inv.summary();

    println!("\n--- Elektronik ---");
    for p in inv.by_category(&Category::Electronics) {
        println!("  {} - Rp{:.0}", p.name, p.price);
    }

    if let Some(product) = inv.find_by_name("Laptop") {
        println!("\nFound: {:?}", product.name);
    }
}
```

### Soal 5 — Option dengan find (Sedang)

Buat struct `StudentList` yang menyimpan `Vec` of `Student` (name, gpa). Implementasikan method `find_best` yang mengembalikan `Option` referensi ke student dengan GPA tertinggi.

**Jawaban:**

```rust
struct Student {
    name: String,
    gpa: f64,
}

struct StudentList {
    students: Vec<Student>,
}

impl StudentList {
    fn find_best(&self) -> Option<&Student> {
        self.students.iter().max_by(|a, b| {
            a.gpa.partial_cmp(&b.gpa).unwrap()
        })
    }
}

fn main() {
    let list = StudentList {
        students: vec![
            Student { name: String::from("Andi"), gpa: 3.5 },
            Student { name: String::from("Budi"), gpa: 3.9 },
            Student { name: String::from("Citra"), gpa: 3.7 },
        ],
    };

    match list.find_best() {
        Some(s) => println!("Terbaik: {} (GPA: {})", s.name, s.gpa),
        None => println!("Tidak ada siswa"),
    }
}
```

### Soal 6 — Nested Enum (Sulit)

Buat enum `Expr` untuk merepresentasikan ekspresi matematika: `Num(f64)`, `Add(Box of Expr, Box of Expr)`, `Mul(Box of Expr, Box of Expr)`. Implementasikan method `eval` yang mengevaluasi ekspresi secara rekursif.

**Jawaban:**

```rust
enum Expr {
    Num(f64),
    Add(Box<Expr>, Box<Expr>),
    Mul(Box<Expr>, Box<Expr>),
}

impl Expr {
    fn eval(&self) -> f64 {
        match self {
            Expr::Num(n) => *n,
            Expr::Add(a, b) => a.eval() + b.eval(),
            Expr::Mul(a, b) => a.eval() * b.eval(),
        }
    }
}

fn main() {
    // (2 + 3) * 4 = 20
    let expr = Expr::Mul(
        Box::new(Expr::Add(
            Box::new(Expr::Num(2.0)),
            Box::new(Expr::Num(3.0)),
        )),
        Box::new(Expr::Num(4.0)),
    );

    println!("Result: {}", expr.eval());
}
```

### Soal 7 — Struct dengan Multiple Impl Blocks (Sedang)

Buat struct `BankAccount` dengan field `owner`, `balance`. Buat dua impl blocks: satu untuk constructor dan query methods, satu lagi untuk mutation methods (deposit, withdraw).

**Jawaban:**

```rust
struct BankAccount {
    owner: String,
    balance: f64,
}

// Query methods
impl BankAccount {
    fn new(owner: &str, initial: f64) -> BankAccount {
        BankAccount {
            owner: String::from(owner),
            balance: initial,
        }
    }

    fn get_balance(&self) -> f64 {
        self.balance
    }

    fn summary(&self) {
        println!("Akun: {} | Saldo: Rp{:.0}", self.owner, self.balance);
    }
}

// Mutation methods
impl BankAccount {
    fn deposit(&mut self, amount: f64) {
        self.balance += amount;
        println!("Deposit Rp{:.0} berhasil", amount);
    }

    fn withdraw(&mut self, amount: f64) -> bool {
        if amount > self.balance {
            println!("Saldo tidak cukup");
            return false;
        }
        self.balance -= amount;
        println!("Withdraw Rp{:.0} berhasil", amount);
        true
    }
}

fn main() {
    let mut acc = BankAccount::new("Budi", 1_000_000.0);
    acc.summary();
    acc.deposit(500_000.0);
    acc.withdraw(200_000.0);
    acc.summary();
}
```

### Soal 8 — State Machine dengan Enum (Sulit)

Buat state machine untuk order: `Pending` -&gt; `Processing` -&gt; `Shipped(tracking)` -&gt; `Delivered`. Implementasikan method `next` yang memajukan state dan `describe` yang mencetak status.

**Jawaban:**

```rust
#[derive(Debug)]
enum OrderState {
    Pending,
    Processing,
    Shipped(String),
    Delivered,
}

struct Order {
    id: u32,
    state: OrderState,
}

impl Order {
    fn new(id: u32) -> Order {
        Order {
            id,
            state: OrderState::Pending,
        }
    }

    fn next(&mut self, tracking: Option<String>) {
        self.state = match &self.state {
            OrderState::Pending => OrderState::Processing,
            OrderState::Processing => {
                let code = tracking.unwrap_or_else(|| String::from("TRK-000"));
                OrderState::Shipped(code)
            }
            OrderState::Shipped(_) => OrderState::Delivered,
            OrderState::Delivered => {
                println!("Order sudah selesai");
                return;
            }
        };
    }

    fn describe(&self) {
        let status = match &self.state {
            OrderState::Pending => String::from("Menunggu"),
            OrderState::Processing => String::from("Diproses"),
            OrderState::Shipped(code) => format!("Dikirim ({})", code),
            OrderState::Delivered => String::from("Terkirim"),
        };
        println!("Order #{}: {}", self.id, status);
    }
}

fn main() {
    let mut order = Order::new(1001);
    order.describe();
    order.next(None);
    order.describe();
    order.next(Some(String::from("JNE-123456")));
    order.describe();
    order.next(None);
    order.describe();
}
```
